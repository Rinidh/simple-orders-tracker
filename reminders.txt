FEATURES 'MUST' INCLUDE after MVP:
- Performace (fast UI animations, fast backend, small bundle size, smaller libraries, )
- All Gujrati text / language switch button

- localStorage saving + retry conn if failed (see gpt 👆)
- Data safety: manual export (JSON/CSV) & cloud backup option
- handles partial payments

APP-USE MIND PLAN:
User gets an order -> User clicks on a button to create a new order -> user sees a bunch of rectangular cards rep orders & their status -> user clicks on any card to know more -> user updates the card when order status changes -> only specific users can update specific things eg cook cannot update card to order paid status or cashier cannot update card to order ready status -> order payment status in card is automatically updated when online payment occurs

EFFICIENT USE PLACE:
- When many cooks/users are managing multiple recipes collectively and any new cook who is delegated to complete the order gets to know what is the current status of the order eg busy kitchens of McDonalds, KFC etc
- When one user is managing multiple orders and needs to know how far each order has progressed


MVP DEVELOPMENT PLAN:
Phase 1: Backend (Weeks 1-2)

Data Models:

Order schema with customer info, snack items, status, payment tracking
Snack schema for item catalog

API Endpoints:

CRUD operations: /api/orders, /api/orders/:id
Status & payment updates: /api/orders/:id/status, /api/orders/:id/payment
Reports: /api/reports/weekly, /api/reports/date-range
Backend Structure:

Models in backend/models/
Route handlers in backend/routes/
Controllers for business logic
Middleware for validation and error handling
Phase 2: Frontend (Weeks 2-3)
Core Pages:

Orders List (main dashboard with filters)
Order Detail (full view with inline edits)
Create Order (form with dynamic snack items)
Reports Summary (weekly/date-range sales totals)
Reusable Components:

StatusBadge, OrderCard, OrderForm, Navbar
Mobile-friendly styling with touch-optimized controls
Frontend Structure:

React Router for navigation
Custom hooks for API calls
Mobile-first responsive CSS
Phase 3: Integration & Deployment (Week 4)
Frontend API integration with backend
Error handling and loading states
Build Vite bundle and serve from backend
Deploy to cloud platform
Success Criteria

PROBLEMATIC CODE:
- order.controller.ts: getRouteParam() returns null if req.params.id is string[]. Small issue as req.params.id is rarely string[]
- current validateItems(value) requires an entire 'array of body.items'. In updateItems(), the frontend will have to send the entire array of items again even though only one item's one field has got updated. Small issue as array of items is not usually very big and frontend can re-send whole array of items with updated item fields
